# 產品需求文件 (PRD)：104 人選自動搜尋與評分排序系統

## 1. 產品概述

建立一套系統性的自動化招募工具，依據公司刊登的職缺 JD（Job Description），定期搜尋 104 人力銀行資料庫中的人選，並依照 JD Scorecard 評分機制對人選進行匹配度排序，協助招募團隊高效篩選合適候選人。

## 2. 目標與範圍

### 目標
- 減少招募人員手動搜尋 104 資料庫的時間
- 透過標準化的 Scorecard 評分機制，提升人選篩選的一致性與準確度
- 建立定期自動化搜尋排程，確保不遺漏潛在優質人選

### 範圍
- 整合公司現有職缺 JD 資訊
- 串接 104 人力銀行資料庫搜尋功能
- 建立 JD Scorecard 評分模型
- 開發定期排程與通知機制

## 3. 核心功能需求

### 3.1 職缺 JD 管理模組
- 匯入 / 同步公司目前刊登中的職缺 JD
- 擷取 JD 中的關鍵條件：職稱、技能需求、工作經驗、學歷要求、產業背景、語言能力等
- 支援手動調整 JD 關鍵條件與權重設定

### 3.2 多平台人才資料庫搜尋模組

#### 3.2.1 104 人力銀行搜尋
- 依據 JD 關鍵條件自動組合搜尋條件
- 呼叫 104 人力銀行資料庫 API 或爬蟲機制進行人選搜尋
- 支援多維度搜尋：職稱、技能、工作經驗年資、學歷、地區、期望薪資等

#### 3.2.2 LinkedIn 人才搜尋
- 串接 LinkedIn Recruiter API / LinkedIn Talent Solutions
- 搜尋維度：職稱、技能標籤、公司經歷、產業、地區、學歷、推薦人脈關係
- 取得公開 Profile 資訊：工作經歷、技能認證（Skills & Endorsements）、推薦信（Recommendations）
- 分析人選活躍度（近期是否更新 Profile、是否標示 Open to Work）
- 支援 Boolean Search 進階搜尋語法

#### 3.2.3 跨平台共用機制
- 自動排除已聯繫 / 已面試 / 已拒絕之重複人選
- 跨平台人選去重（同一人在 104 與 LinkedIn 皆有資料時合併為單一候選人）
- 統一人選資料格式，供 Scorecard 模組進行一致性評分
- 標註人選資料來源平台，方便後續聯繫時選擇適當管道

### 3.3 JD Scorecard 評分與排序模組
- 依據 JD 定義的各項條件建立 Scorecard 評分卡
- 評分維度（可自訂權重）：
  - 技能匹配度
  - 工作經驗相關性
  - 學歷符合度
  - 產業背景相關性
  - 地區 / 到職意願
  - 薪資期望匹配度
- 依照總分進行人選排序，產出排名清單
- 支援篩選門檻設定（如：總分需達 70 分以上）

### 3.4 JD 優化反饋機制模組
- 根據人選搜尋結果與 Scorecard 評分數據，反向分析 JD 的有效性
- 反饋維度：
  - **搜尋命中率分析**：若符合條件的人選數量過少，建議放寬特定條件（如降低經驗年資要求、擴大技能關鍵字）
  - **條件衝突偵測**：識別 JD 中互相矛盾或不合理的組合條件（如：要求 10 年經驗但薪資低於市場行情）
  - **市場供需比對**：比對 104 資料庫中符合條件的人選總量，評估該職缺在人才市場的競爭力
  - **高分人選特徵回饋**：分析高分人選的共同特徵，建議 JD 調整方向以吸引更多類似人才
  - **歷史招募成效追蹤**：追蹤經由系統推薦且最終錄取的人選，回饋哪些 Scorecard 維度最具預測力
- 產出 JD 優化建議報告，包含具體修改建議與預期效果
- 支援一鍵套用建議，自動更新 JD 關鍵條件與權重
- 定期產出 JD 健康度評分（JD Health Score），協助招募團隊持續優化職缺描述

### 3.5 定期排程與通知模組
- 支援自訂搜尋排程頻率（每日 / 每週 / 自訂）
- 每次搜尋後自動產出新增人選報告
- 透過 Email / Slack / 系統通知推送搜尋結果給招募負責人
- 記錄歷次搜尋結果，支援趨勢分析

## 5. 非功能需求

- **效能**：單次搜尋應於 5 分鐘內完成（含評分排序）
- **可靠性**：排程任務失敗時自動重試，並通知管理員
- **安全性**：人選個資依照個資法規範處理與儲存
- **擴展性**：未來可擴展至其他人力銀行平台（如 1111、CakeResume、Glassdoor）

## 6. 成功指標

- 招募人員每週搜尋 104 的時間減少 60% 以上
- 人選初篩準確率提升至 80% 以上
- 從職缺刊登到首批人選推薦的時間縮短至 24 小時內
